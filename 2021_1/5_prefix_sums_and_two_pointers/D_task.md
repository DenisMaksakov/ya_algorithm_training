# D. Город Че

|                     |                                  |
| ------------------- | -------------------------------- |
| Ограничение времени | 1 секунда                        |
| Ограничение памяти  | 64Mb                             |
| Ввод                | стандартный ввод или input.txt   |
| Вывод               | стандартный вывод или output.txt |

В центре города Че есть пешеходная улица - одно из самых популярных мест для прогулок жителей города. По этой улице очень приятно гулять, ведь вдоль улицы расположено $n$ забавных памятников.

Девочке Маше из города Че нравятся два мальчика из ее школы, и она никак не может сделать выбор между ними. Чтобы принять окончательное решение, она решила назначить обоим мальчикам свидание в одно и то же время. Маша хочет выбрать два памятника на пешеходной улице, около которых мальчики будут ее ждать. При этом она хочет выбрать такие памятники, чтобы мальчики не увидели друг друга. Маша знает, что из-за тумана мальчики увидят друг друга только в том случае, если они будут на расстоянии не более $r$ метров.

Маше заинтересовалась, а сколько способов есть выбрать два различных памятника для организации свиданий.

## Формат ввода

В первой строке входного файла находятся два целых числа $n$ и $r$ ($2≤n≤300000$, $1≤r≤10^9$) - количество памятников и максимальное расстояние, на котором мальчики могут увидеть друг друга.

Во второй строке задано $n$ положительных чисел $d_1​,…,d_n$​, где $d_i$​ - расстояние от $i$-го памятника до начала улицы. Все памятники находятся на разном расстоянии от начала улицы. Памятники приведены в порядке возрастания расстояния от начала улицы ($1≤d_1,d_2,…,d_n≤10^9$).

## Формат вывода

Выведите одно число - число способов выбрать два памятника для организации свиданий.

## Пример
Ввод
```
4 4
1 3 5 8
```
Вывод
```
2
```

## Примечания

В приведенном примере Маша может выбрать памятники 1 и 4 или памятники 2 и 4.

## Решение

**Алгоритм "два указателя" для подсчета пар с расстоянием > r**

### Идея алгоритма:
- Используем два указателя: `left` (начало пары) и `right` (конец пары)
- Так как массив отсортирован по расстоянию, при движении `right` вправо минимальное расстояние увеличивается

### Шаги алгоритма:
1. **Инициализация**: `left = 0`, `right = 1`
2. **Для каждого `left`** находим первый `right`, где `distance[right] - distance[left] > r`
3. **Подсчет**: Все пары `(left, right), (left, right+1), ..., (left, n-1)` - валидные
4. **Формула**: `count += n - right`

### Сложность:
- **Время**: O(n) - каждый указатель проходит массив один раз
- **Память**: O(1) - используем только константную дополнительную память

### Пример для `[1, 3, 5, 8]`, `r = 4`:
```
left=0: right=2 (5-1=4≤4) → right=3 (8-1=7>4) → count += 4-3=1
left=1: right=3 (8-3=5>4) → count += 4-3=1
left=2: right=3 (8-5=3≤4) → нет добавления
Итого: 2 пары
```

**Результат**: Эффективно находит все пары памятников на расстоянии > r без полного перебора $O(n²)$.
